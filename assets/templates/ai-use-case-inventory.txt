AI USE CASE INVENTORY WORKSHEET
Zero → Frontier — free, ungated, no attribution required

Fill in one row per recurring task you do. This surfaces where AI
assistance is actually worth trying versus where it isn't.

For each task, answer:

TASK NAME:
  e.g. "Weekly status update to my team"

HOW OFTEN:
  e.g. daily / weekly / monthly

CURRENT TIME COST:
  e.g. 30 minutes per instance

WHAT MAKES IT SLOW:
  e.g. "I know what to say, drafting it takes the time" vs.
  "I have to research the answer first" — these need different
  approaches. Drafting-bottleneck tasks are good AI candidates.
  Research-bottleneck tasks need real sources, not guessing.

CONFIDENTIALITY LEVEL:
  Public / Internal / Confidential / Highly sensitive
  (Anything above "Internal" — check your tool's data-handling
  terms before pasting it in. See the Tools page.)

REVERSIBILITY:
  Fully reversible (a draft you'll reread) vs. hard to reverse
  (a sent email, a filed document, a public post)
  Hard-to-reverse tasks always need a human review step.

SUGGESTED APPROACH:
  Which Use Case pattern this matches (meeting notes, research
  synthesis, drafting, code help, etc.) and which tool fits best.

WORTH TRYING? (yes / no / not yet)
